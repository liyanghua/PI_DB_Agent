# 元策略定义

**业务场景及任务维度：客户业务专家**

| **维度** | **释义说明** | **示例** |
| --- | --- | --- |
| 业务场景 | 明确业务所属场景 | 商品链接优化 |
| 场景目标 | 定义本次任务要达成的业务结果 | 发现问题和机会，形成改进动作 |
| 输出成果 | 本次任务交付的可落地产物 | 改进方案建议 |
| 流程图或业务模型 | 支撑优化动作的底层逻辑与诊断模型 | 商品诊断维度拆解：流量结构诊断、标签匹配度诊断、产品定位诊断、视觉诊断（主图/详情诊断）、营销推广诊断 |
| 执行步骤 | 按照思维框架拆解后各维度的重要性程度进行执行相关动作，明确每一步动作 | 1.数据初诊：确认转化率与行业均值差距  <br>2.流量结构诊断：分析流量来源、占比、流量漏斗、停留时长等  <br>3.标签与定位诊断：标题关键词、类目、人群标签匹配度  <br>4.内容视觉诊断：主图、详情页逻辑、卖点、信任背书、购买顾虑  <br>5.优化动作落地：按模块修改  <br>6.复盘迭代：监控数据，沉淀优化经验 |
| 关键节点思路说明（为什么这么做）【待定】 | 解释每个核心步骤背后的业务逻辑，明确动作必要性 | 对于流量结构诊断：  <br>先判断是流量不精准还是承接能力不足，避免盲目改详情页——判定是流量不精准  <br>分析是否是流量来源渠道导致不精准——判定是  <br>分析搜索流量还是 |
| 页面截图 | 流程涉及的关键后台/操作页面 | 流量分析盘、转化数据漏斗截图 |
| 常见问题类型 | 优化过程中容易出现的问题/机会，及具体表现是什么 | 流量不精准：  <br>用户标签与商品标签不符，自然推流不准  <br>付费推广直通车/搜索圈选人群不精准  <br>结构占比出现偏向导致整体不精准 |
| 判断依据/指标 | 用于诊断问题、验证效果的核心数据指标 | 商品转化率CVR、UV价值、跳失率、平均停留时长 |
| 判断标准（数量/质量/时限/频率） | 指标高低好坏的量化阈值与对比标准 | 类目商品转化率CVR以行业均值为基准：  <br>差：CVR＜行业均值的80%  <br>合格：行业均值的80%≤CVR＜行业均值的100%  <br>优秀：CVR≥行业均值 |
| 问题解决方法/执行动作 | 针对问题类型的具体优化动作 | 流量不精准：  <br>优化标题关键词、类目、属性等强化链接标签  <br>付费推广圈选人群标签精准化，关键词、行业词、关联词的精准性 |
| 流程触发与终止条件 | 优化流程启动与结束的边界条件 | 触发条件：如，日访客≥100且CVR＜行业均值的80%，持续≥5天  <br>终止条件：CVR提升至行业均值以上，或连续优化后CVR提升≤0.5%，维持就好 |
| 方法是否有效的验证方式 | 通过什么数据/方式证明优化动作有效 | CVR提升≥1个百分点，或超越行业均值 |
| 工具表单模板 | 执行流程中可复用的工具/表单 | 商品链接优化诊断表、商品链接各维度优化方案 |

**经营增长目标提升维度：**

| 包含维度 | 核心逻辑 | 释义说明 | 示例 |
| --- | --- | --- | --- |
| 1. 目标 | 这个方法论到底要解决什么问题？ | 不能只讲理念，必须有明确结果 | 在搜索曝光量不变的情况下，商品主图点击率从3%提升至5%，从而拉动搜索流量增长 |
| 2. 对象 | 它研究的对象是谁？ | 要说清楚是人、货、场、流量、内容、组织还是系统 | 搜索场景下的商品主图 |
| 3. 场景 | 它适合用在什么情况下？ | 不能什么都能用，必须有适用场景 | 商品具有稳定搜索曝光（如≥1000），但因主图点击率低导致流量获取效率低下 |
| 4. 问题分类 | 容易出现哪几类问题 | 遇到问题时，能先判断属于哪一类。针对要分析的对象，逐个判断 | ①主图卖点与用户搜索关键词不匹配  <br>②主图模糊、无差异，无法打动用户  <br>③主图中信息密度不合理，用户无法快速抓取关键信息  <br>④主图构图视觉杂乱、画质差、缺乏视觉焦点 |
| 5. 核心变量 | 它主要看哪几个关键因素？ | 数据指标名称（维度名称） | ②主图较模糊  <br>a搜索点击率CTR  <br>b用户在主图上的视觉焦点数据值  <br>c不同清晰度的主图版本测图数据表现 |
| 6. 判断指标 | 用什么数据或现象判断？ | 数量/质量/频次/时限等数值 | a搜索点击率（CTR）行业均值为4%  <br>差：CTR＜行业均值的80%  <br>合格：行业均值的80% ≤CTR＜行业均值的100%  <br>优秀：CTR≥行业均值 |
| 7. 判断标准 | 高低好坏怎么定？ | 必须有阈值、等级、对比标准 | 3%较3.2%低，即搜索点击率（CTR）差 |
| 8. 因果关系 | 为什么这个指标说明这个问题？ | 不能只说结论，要能解释原因 | CTR低  <br>主图模糊，用户看到后观感差没有吸引力，用户会快速划走 |
| 9. 决策顺序 | 先判断什么，再判断什么？ | 要有顺序，不能东看一眼西看一眼 | CTR数据低于行业均值，确认问题存在：  <br>评估主图视觉呈现：清晰度、构图、信息密度是否合理  <br>拆解主图中卖点是否符合用户需求  <br>分析低CTR的核心关键词来判断匹配度 |
| 10. 建议动作 | 判断完以后今天干什么？ | 不能只诊断，不给动作 | 制作3-5个不同清晰度主图，进行A/B测试  <br>优化主图结构，减少干扰元素，放大核心产品与卖点  <br>突出1-2个差异化卖点  <br>提高图中高流量吸睛词 |
| 11. 执行条件 | 什么情况下才能执行这个方案？ | 要知道前提，不是所有店都能照搬 | 美工或AI生成工具  <br>主图修改与测图能力  <br>清晰行业均值  <br>商品曝光大于1000，有研究价值 |
| 12. 例外情况 | 什么情况下这个判断不成立？ | 要有边界，避免误判 | 如，展现量过低（＜100），CTR不具备统计意义  <br>平台规则变化，在主图审核、流量分发机制发生调整 |
| 13. 验证方式 | 看什么证明该动作有效？ | 要有复盘指标 | 优化后主图点击率提升≥2个百分点，达到5% |
