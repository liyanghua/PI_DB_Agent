---

# 一、业务场景

适用于淘天电商中所有需要做市场判断、选品判断、产品开发、链接规划、竞品定位、价格带布局、视觉策划的业务场景。

具体包括：

1.  淘宝 / 天猫新品开发
    
2.  爆款机会挖掘
    
3.  类目市场分析
    
4.  主图与视觉策划
    
5.  竞品分析
    
6.  价格带布局
    
7.  产品升级与迭代
    
8.  跨平台趋势机会挖掘
    
9.  季节性产品提前布局
    
10.  店铺产品结构规划
    

---

# 二、场景目标

市场洞察不是为了写报告，而是为了回答 8 个经营问题：

1.  **这个类目现在什么东西卖得好？**  
    找到行业主流产品、主流款式、主流价格带、主流卖点。
    
2.  **这个类目什么东西涨得快？**  
    找到趋势产品、趋势关键词、高增速新品、高增速店铺。
    
3.  **用户到底在搜什么？**  
    通过关键词判断需求结构、人群需求、场景需求、功能需求、属性需求。
    
4.  **用户真正关心和抱怨什么？**  
    通过评价、问大家找到购买顾虑、使用痛点、产品升级点。
    
5.  **哪些地方竞争没有那么强？**  
    找到对手没有做、对手少、对手弱的机会。
    
6.  **我们应该做什么产品？**  
    明确产品方向、款式方向、功能方向、材质方向、价格带方向。
    
7.  **我们怎么和竞品打？**  
    明确竞品是谁、竞品强在哪里、弱在哪里、我们差异化切入点是什么。
    
8.  **最后能不能落到链接规划？**  
    输出新品开发表、链接定位表、价格带布局表、主图策划表、产品升级表。
    

一句话总结：

**市场洞察的目标，是用数据找到“有需求、有增长、有利润、竞争可切入”的产品机会。**

# 三、输出成果

完成一次标准市场洞察后，必须输出 10 类成果。

| 输出成果 | 作用 | 最终要回答的问题 |
| --- | --- | --- |
| 1. 类目大盘分析表 | 判断行业主流与趋势 | 现在市场卖什么？什么涨得快？ |
| 2. 行业前300商品分析表 | 找爆款共性 | 爆款共同具备什么属性？ |
| 3. 关键词需求拆解表 | 拆用户需求 | 用户在搜索什么需求？ |

| 5. 评价/问大家痛点表 | 找产品升级点 | 用户购买前顾虑和购买后不满是什么？ |

# 四、整体流程

完整流程分为 10 步：

流程1：确定分析边界  
流程2：行业大盘与热销商品分析  
流程3：爆款基因提炼  
流程4：关键词需求分析  
流程5：评价与问大家痛点分析  
流程6：价格带与利润空间分析  
流程7：竞品与竞店格局分析  
流程8：跨平台趋势机会分析  
流程9：机会判断与产品创新设计  
流程10：产品开发与链接规划落地

整体逻辑是：

**先看全域趋势，再看行业结构，再看竞品竞店，再看用户需求，再看价格空间，最后落到产品和链接规划。**

# 五、具体流程及执行步骤

## 流程1：确定分析边界

### 1.1 目的

先明确“分析对象是谁”，避免市场洞察变成泛泛而谈。

很多运营分析市场时容易犯三个错误：

第一，只盯一个竞品单品；  
第二，只看本行业，不看全网趋势；  
第三，没有先确定自己的产品、价格、人群、店铺阶段，导致分析出来的机会无法落地。

### 1.2 执行动作

先填写《市场洞察项目定义表》。

| 字段 | 填写要求 |
| --- | --- |
| 分析类目 | 精确到三级类目或最小叶子类目 |
| 分析产品线 | 例如沙发垫、桌垫、防晒衣、假发、地垫 |
| 店铺阶段 | 新店 / 成长期店铺 / 爆款店铺 / 多链接店铺 |
| 当前目标 | 新品开发 / 爆款挖掘 / 产品升级 / 竞品突破 |
| 当前资源 | 供应链、价格、视觉、投放、品牌、客服等优势 |
| 目标价格带 | 低价 / 中价 / 高价 / 生意参谋6个价格带 |
| 目标人群 | 已知人群 / 待分析人群 |
| 分析周期 | 近7天 / 近30天 / 月维度 / 季节维度 |

### 1.3 判断标准

| 判断项 | 可落地标准 |
| --- | --- |
| 类目是否清楚 | 必须精确到三级类目或最小可比类目 |
| 产品是否清楚 | 必须明确是做新品、老品升级，还是价格带补位 |
| 分析周期是否清楚 | 趋势看7天/30天，主流结构看月度，季节品看去年同期和当季 |
| 目标是否清楚 | 最终必须落到“做什么产品/怎么定价/怎么表达/打谁” |

### 1.4 产出

《市场洞察项目定义表》。

---

## 流程2：行业大盘与热销商品分析

### 2.1 目的

判断这个类目当前的主流产品、主流价格、主流卖点、主流视觉和趋势变化。

《市场洞察报告》分析市场的目的是了解“谁涨得快、谁稳定、谁赚钱”，并通过行业数据判断流行趋势和机会点。

### 2.2 数据来源

1.  生意参谋：市场 → 市场排行 → 三级类目 → 商品排行榜
    
2.  店透视：一键分析 → 加载全部 → 导出表格
    
3.  边界BI：市场洞察 → 类目排行榜
    
4.  行业前300商品数据
    
5.  如果是款式类目，可先取行业前50或前100款做款式分析
    

### 2.3 执行动作

第一步：导出行业前300商品榜单。  
第二步：删除无效列，例如日期、类目名称、图片链接。  
第三步：把商品主图嵌入单元格，方便横向对比。  
第四步：逐个补充商品属性字段。  
第五步：按产品类型、价格带、材质、功能、场景、风格、卖点做分类统计。  
第六步：找出高销量、高增长、高客单、高利润、低竞争的商品。

### 2.4 表格字段

| 字段 | 说明 |
| --- | --- |
| 排名 | 行业排名 |
| 店铺名 | 对手是谁 |
| 商品链接 | 分析对象 |
| 商品主图 | 看视觉表达 |
| 销量/支付买家数 | 判断真实销售能力 |
| GMV/交易指数 | 判断体量 |
| 客单价 | 判断价格带 |
| 价格带 | 低/中/高（生意参谋6个价格带） |
| 产品类型 | 品类/款式/形态 |
| 材质 | 例如半边绒、冰丝、羊羔绒 |
| 功能 | 防水、防滑、抗菌、护眼、显白等 |
| 风格 | 奶油风、轻奢风、复古风、简约风 |
| 场景 | 家用、宿舍、母婴、户外、通勤 |
| 主卖点 | 第一卖点是什么 |
| 主图元素 | 背景、构图、文案、场景、人物 |
| 是否高增速 | 近7天/近30天排名提升明显 |
| 爆款原因 | 为什么卖得好 |

### 2.5 分析结论模板[《7个结论的判断详细流程》](https://alidocs.dingtalk.com/i/nodes/7dx2rn0JbY0apqaXt2Ll61aBVMGjLRb3)

完成表格后，不要只输出数据，要输出 7 个结论：

1.  当前行业热卖产品分为哪几类？
    
2.  每类产品的市场占比大概是多少？
    
3.  行业主流价格带是什么？
    
4.  高价、中价、低价分别卖什么人群？
    
5.  行业爆款共性是什么？
    
6.  行业差异机会在哪里？
    
7.  哪些产品涨得快，原因是什么？

## 流程3：爆款基因提炼

完整文档：[《爆款基因提炼》](https://alidocs.dingtalk.com/i/nodes/4lgGw3P8vRqw1ow9spd3PXqr85daZ90D)

### 3.1 目的

从热销商品中提炼“可复制的爆款规律”，不是简单抄款，而是拆出爆款背后的结构。

爆款基因不是一个点，而是由以下因素共同构成：

**产品类型 + 材质 + 功能 + 风格 + 人群 + 场景 + 价格带 + 视觉表达 + 流量入口。**

### 3.2 执行动作

从行业前50、前100或前300商品中，统计出现频率最高的元素。

### 3.3 爆款基因分析维度

| 维度 | 分析什么 | 明确判断标准 |
| --- | --- | --- |
| 产品形态 | 热销的是标准款、异形款、组合款、套装款、升级款？ | ① 在行业TOP50商品中，某形态出现≥15个，占比≥30%，判定为“主流爆款形态”；② TOP100中占比仍≥20%，判定为“稳定爆款形态”；③ TOP50中出现5-14个，但其中近30天排名提升≥20名的商品占比≥30%，判定为“趋势形态”；④ TOP50中少于5个，暂不作为主流爆款形态 |
| 材质 | 哪些材质卖得最多？ | ① 某材质在TOP50中出现≥15个，占比≥30%，判定为“主流热销材质”；② TOP100中占比≥20%，说明该材质稳定；③ 高增速商品中某材质占比≥30%，判定为“趋势材质”；④ 评价/问大家中材质相关好评或顾虑占比≥15%，说明材质是核心购买决策因素 |
| 功能 | 用户主要为什么买？ | ① 某功能词在关键词TOP300中搜索人气占比≥10%，判定为“核心功能需求”；② TOP50商品标题/主图/详情页中出现≥15次，说明商家普遍主打该功能；③ 问大家中该功能顾虑占比≥15%，说明用户购买前高度关注；④ 差评中该功能问题占比≥10%，说明存在功能升级机会 |
| 风格 | 哪些风格更容易爆？ | ① 某风格在TOP50中出现≥12个，占比≥24%，判定为“主流风格”；② 高增速商品中该风格占比≥25%，判定为“趋势风格”；③ 该风格词搜索增长≥20%，说明搜索需求正在上升；④ 小红书500赞以上内容或抖音1000赞以上视频中高频出现，说明具备跨平台趋势验证 |
| 人群 | 谁在买？ | ① 某人群词在关键词TOP300中占比≥8%，判定为“明确人群需求”；② TOP50中该人群定位商品占比≥20%，判定为“主流购买人群”；③ 高增速商品中该人群占比≥30%，判定为“趋势人群”；④ 标题、主图、详情页、评价、问大家、达摩盘中至少3类证据一致，才能确认该人群标签 |
| 场景 | 用在什么场景？ | ① 某场景词在关键词TOP300中占比≥8%，判定为“重要场景需求”；② TOP50主图中明确表达该场景的商品≥12个，判定为“主流成交场景”；③ 评价/问大家中该场景相关内容占比≥10%，说明真实使用场景成立；④ 场景词搜索增长≥20%，判定为“趋势场景”；⑤ 有搜索需求但TOP50中场景表达少于5个，判定为“场景差异机会” |
| 价格带 | 爆款集中在哪个价格区间？ | ① 某价格带在TOP50中商品数量占比≥30%，判定为“爆款集中价格带”；② 该价格带支付买家数占比≥30%，判定为“主流成交价格带”；③ 该价格带GMV占比≥30%，判定为“主流金额价格带”；④ 商品数占比＜15%，但支付买家数占比≥25%，判定为“供给不足的机会价格带”；⑤ GMV占比高但买家数占比低，判定为“高客单利润价格带” |
| 视觉 | 主图共性是什么？ | ① TOP50中某种主图结构、背景、构图、卖点表达出现≥20个，占比≥40%，判定为“主流视觉结构”；② 同类背景/构图/卖点表达占比≥70%，判定为“高度同质化”，存在视觉差异机会；③ 高增速商品中某视觉结构占比≥30%，判定为“趋势视觉表达”；④ TOP10中≥6个使用同一视觉结构，说明头部视觉共性强 |
| 增长原因 | 为什么能起来？ | 至少满足以下2项，才能判断增长原因成立：① 近30天排名提升≥20名；② 近7天支付买家数增长≥30%；③ 近30天支付买家数增长≥50%；④ 相关关键词搜索增长≥20%；⑤ 小店/C店也能进入TOP50，说明产品机会大于品牌优势；⑥ 跨平台内容热度高，且淘宝供给少；⑦ 竞品少、视觉弱、价格高或差评集中，说明存在可突破空间 |

### 3.4 产出公式

每一个爆款基因都要按这个格式输出：

**爆款基因 = 高频市场元素 + 明确需求场景 + 可证明卖点 + 合理价格带 + 竞品可突破点**

例如沙发垫：

**半边绒四季面料 + 黑白条纹设计 + 现代轻奢客厅场景 + 中价位大众价格带 + 主图风格统一 = 可开发款式方向。**

### 3.5 可执行判断

#### 根据爆款基因强度分级

| 等级 | 判断标准 | 业务动作 |
| --- | --- | --- |
| 强爆款基因 | 满足：TOP50占比≥30%，TOP100占比≥20%，支付买家数占比≥30%，GMV占比≥30% 中任意2项 | 可作为主推款、新品开发、爆款复制方向 |
| 趋势爆款基因 | 满足：高增速商品占比≥30%，关键词搜索增长≥20%，近30天支付买家数增长≥50%，跨平台内容高热 中任意2项 | 适合快速测款、小批量上新、抢趋势红利 |
| 差异机会基因 | 满足：评价痛点占比≥10%，问大家顾虑占比≥10%，TOP50承接商品少于5个，价格带供给占比＜15%但买家数占比≥25% 中任意2项 | 适合做产品升级、视觉差异化、价格带错位 |
| 不建议作为爆款基因 | TOP50出现少于5个，关键词无增长，评价/问大家无高频需求，支付买家数无集中 | 暂不开发，只做观察 |

## 流程4：关键词需求分析

完整文档：[《关键词分析后的 8 个必输出结论与判断标准》](https://alidocs.dingtalk.com/i/nodes/G53mjyd80pBgkdgZigdyd27E86zbX04v)

### 4.1 目的

关键词不是简单的流量词，而是用户需求的直接表达。

《市场洞察报告》关键词是需求属性的精准表达，也是流量入口分化度的标志；通过关键词需求表，可以判断趋势市场、趋势赛道，以及产品应该做大爆款、小爆款还是结构动销款。

### 4.2 数据来源

1.  生意参谋关键词
    
2.  边界BI关键词
    
3.  店透视关键词下载
    
4.  淘宝搜索下拉词
    
5.  商品360标题优化
    
6.  竞品标题词根
    
7.  搜索榜、飙升词、新增词
    

### 4.3 执行动作

第一步：下载类目TOP300关键词。  
第二步：拆词根。  
第三步：把词根归入八大需求类型。  
第四步：统计每类需求占比。  
第五步：按搜索人气、增长率、竞争度筛选TOP20词根。  
第六步：判断产品机会。

### 4.4 八大需求分类

#### 详细方法参考文档[《关键词词根8大需求聚类提示词》](https://alidocs.dingtalk.com/i/nodes/7dx2rn0JbY0apqaXt25ra305VMGjLRb3)

| 需求类型 | 说明 | 示例 |
| --- | --- | --- |
| 品类需求 | 用户在找什么品 | 桌布、沙发垫、假发、防晒衣 |
| 人群需求 | 用户为谁买 | 老人、儿童、学生、宝妈、男士 |
| 属性需求 | 用户要什么属性 | 大码、长袖、真丝、透明、厚款 |
| 功能需求 | 用户要解决什么问题 | 防水、防滑、防晒、遮肉、抗菌 |
| 场景需求 | 用户在哪用 | 宿舍、客厅、军训、通勤、露营 |
| 品牌需求 | 用户指定品牌 | 某品牌词 |
| 风格需求 | 用户要什么审美 | 轻奢、高级感、奶油风、多巴胺 |
| 定制需求 | 用户有特殊要求 | 定制尺寸、定制图案、异形 |

### 4.5 关键词拆解方法

例如：

| 原关键词 | 拆解词根 | 需求类型 |
| --- | --- | --- |
| 桌布 | 桌布 | 品类需求 |
| 茶几桌垫 | 茶几 / 桌垫 | 场景需求 + 品类需求 |
| 桌布2025新款轻奢高级感 | 桌布 / 2025 / 新款 / 轻奢 / 高级感 | 品类 + 趋势 + 风格 |
| 餐桌垫防油防水防烫高级感 | 餐桌垫 / 防油 / 防水 / 防烫 / 高级感 | 场景 + 功能 + 风格 |

### 4.6 判断标准

| 关键词现象 | 代表含义 | 落地动作 |
| --- | --- | --- |
| 搜索人气高，占比高 | 主流需求 | 做主推款或爆款 |
| 搜索人气高，竞争强 | 大盘词 | 需要产品力、视觉力、价格力 |
| 搜索人气中，增长快 | 趋势机会 | 快速开发测试 |
| 搜索人气小，转化强 | 长尾利润机会 | 做精准款、利润款 |
| 人群词增长快 | 人群机会 | 做极致人群定位 |
| 场景词增长快 | 场景机会 | 改主图、详情页、标题 |
| 功能词增长快 | 产品升级机会 | 开发功能升级款 |
| 风格词增长快 | 视觉和款式机会 | 做款式创新和主图风格 |

### 4.7 产出结论模板：

关键词分析后必须输出：

1.  当前类目最大需求是什么？
    
2.  增长最快的需求是什么？
    
3.  哪些需求我们已有产品承接？
    
4.  哪些需求我们没有产品承接？
    
5.  哪些词适合做大爆款？
    
6.  哪些词适合做小爆款？
    
7.  哪些词适合做利润款或动销款？
    
8.  标题、主图、产品开发分别怎么调整？

## 流程5：评价与问大家痛点分析

完整文档：[《完整评价 / 问大家反馈判断标准》](https://alidocs.dingtalk.com/i/nodes/OG9lyrgJPzvLo2LRczByBX43WzN67Mw4)

### 5.1 目的

关键词告诉你用户“想买什么”，评价和问大家告诉你用户“为什么犹豫、为什么不满、为什么复购或差评”。

### 5.2 数据来源

1.  同类型排名前10竞品的问大家
    
2.  同类型排名前10竞品的评价
    
3.  差评、中评、追评
    
4.  SKU评价
    
5.  买家秀
    
6.  客服高频问题
    
7.  自己店铺售后原因
    

### 5.3 执行动作

第一步：确定同类型竞品前10。  
第二步：下载问大家内容。  
第三步：下载评价内容，重点看中差评和追评。  
第四步：用AI做初步归类。  
第五步：人工复核高频痛点，避免AI归类过粗。  
第六步：计算关注点和痛点占比。  
第七步：形成产品升级和卖点包装建议。

### 5.4 问大家分析表

| 字段 | 说明 |
| --- | --- |
| 竞品链接 | 哪个竞品 |
| 问题原文 | 用户问了什么 |
| 问题类型 | 材质/尺寸/气味/安装/适用/安全/售后 |
| 用户顾虑 | 怕什么 |
| 出现次数 | 高频程度 |
| 占比 | 是否主流顾虑 |
| 竞品回答 | 对手怎么解释 |
| 我方动作 | 主图说明/详情页解释/客服话术/产品升级 |

### 5.5 评价痛点表

| 字段 | 说明 |
| --- | --- |
| 竞品链接 | 来源 |
| 评价原文 | 用户真实反馈 |
| 正/负向 | 好评/中评/差评 |
| 痛点分类 | 材质、气味、尺寸、色差、掉毛、做工、安装 |
| 痛点强度 | 高/中/低 |
| 出现次数 | 频率 |
| 是否可解决 | 可解决/不可解决/需供应链 |
| 产品动作 | 改材质、改尺寸、改包装、改工艺 |
| 表达动作 | 主图、详情页、标题、客服话术 |

### 5.6 判断标准

| 用户反馈 | 说明 | 动作 |
| --- | --- | --- |
| 高频问大家 | 购买前顾虑 | 主图/详情页提前解释 |
| 高频差评 | 产品硬伤 | 产品升级优先级最高 |
| 高频好评 | 用户认可点 | 放大成主卖点 |
| 竞品没解决的问题 | 差异化机会 | 做功能升级或服务承诺 |
| 用户反复问但竞品没讲清楚 | 视觉表达机会 | 主图、详情页重点表达 |

### 5.7 输出公式

**产品升级机会 = 高频痛点 + 竞品未解决 + 我方可改进 + 用户愿意为此买单**

例如：

如果竞品沙发垫差评集中在“滑、不贴合、边角翘、清洗麻烦”，那产品升级方向就不是泛泛写“高品质”，而是：

1.  防滑底升级
    
2.  异形尺寸适配
    
3.  边角包边工艺
    
4.  可机洗材质
    
5.  主图直接打“坐下不移位”

## 流程6：价格带与利润空间分析

### 完整[《价格带市场结构判断标准》](https://alidocs.dingtalk.com/i/nodes/np9zOoBVBYByodygtLGvAy6BW1DK0g6l)

### 6.1 目的

价格带分析不是看“卖多少钱”，而是看：

1.  哪个价格带有流量？
    
2.  哪个价格带竞争强？
    
3.  哪个价格带利润好？
    
4.  哪个价格带适合我们切入？
    
5.  不同价格带对应什么人群？
    

### 6.2 数据来源

1.  生意参谋市场排行
    
2.  边界BI战场沙盘
    
3.  店透视SKU价格
    
4.  竞品SKU销售占比
    
5.  自己产品成本和毛利表
    

### 6.3 执行动作

第一步：把行业TOP300商品按价格排序。  
第二步：划分低价、中价、高价、超高价价格带。  
第三步：统计每个价格带的商品数量、销量、GMV、客单价、品牌数量。  
第四步：看每个价格带的竞品强度。  
第五步：结合自己成本核算毛利。  
第六步：确定切入价格带。

### 6.4 价格带机会表

| 价格带 | 商品数 | 销量占比 | GMV占比 | 客单价 | 主要人群 | 竞争强度 | 毛利空间 | 机会判断 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 低价带 |  |  |  |  | 价格敏感 | 高/中/低 | 低 | 引流款 |
| 中价带 |  |  |  |  | 大众人群 | 高/中/低 | 中 | 主推款 |
| 高价带 |  |  |  |  | 品质人群 | 高/中/低 | 高 | 利润款 |
| 空白价格带 |  |  |  |  | 待验证 | 低 | 待测 | 机会款 |

### 6.5 判断标准

| 市场结构 | 策略 |
| --- | --- |
| 市场都低价 | 可以做高价品质差异，前提是产品和视觉能支撑 |
| 市场都高价 | 可以做中低价高性价比切入 |
| 高中低都有 | 找竞争最弱、利润能承接的价格带 |
| 某价格带商品少但搜索需求高 | 优先测试 |
| 某价格带销量高但毛利低 | 适合引流，不适合利润款 |
| 某价格带GMV高但竞品弱 | 优先开发主推款 |

### 6.6 落地动作

每一个价格带都要匹配产品角色：

| 价格带 | 产品角色 | 运营动作 |
| --- | --- | --- |
| 低价带 | 流量款 | 拉点击、拉访客、做入口 |
| 中价带 | 主推款 | 承接大盘需求，打爆款 |
| 高价带 | 利润款 | 强化材质、功能、品牌、服务 |
| 空白带 | 机会款 | 小批量测款，验证需求 |

## 流程7：竞品与竞店格局分析

完整文档：[《竞品现象判断标准》](https://alidocs.dingtalk.com/i/nodes/7QG4Yx2JpLAO1oOycgYvqyePJ9dEq3XD)

### 7.1 目的

竞品分析不是为了模仿，而是为了找到“我能打谁、怎么打、打哪里”。

### 7.2 找竞品的前提

不能默认行业第一就是竞品。

真正的竞品必须满足：

1.  同一个最小叶子类目
    
2.  同类型产品
    
3.  同价格带或相邻价格带
    
4.  同人群或相似人群
    
5.  同使用场景
    
6.  店铺层级差距不能过大
    
7.  流量入口相似
    
8.  当前阶段打得动
    

### 7.3 执行动作

第一步：从行业商品排行榜筛选同类型产品。  
第二步：按价格带筛选。  
第三步：按场景和人群筛选。  
第四步：按店铺层级筛选。  
第五步：按视觉和卖点筛选。  
第六步：确定3类竞品：直接竞品、学习竞品、防御竞品。

### 7.4 竞品分类

| 竞品类型 | 定义 | 用途 |
| --- | --- | --- |
| 直接竞品 | 同类目、同价格、同人群、同场景 | 直接对标和打击 |
| 学习竞品 | 比我们强，但阶段可学习 | 学习卖点、视觉、SKU、流量 |
| 防御竞品 | 可能抢我们用户 | 监控价格、活动、投放、上新 |
| 趋势竞品 | 增速快的新链接 | 判断新机会 |
| 弱竞品 | 小店、C店、视觉差但卖得好 | 优先突破对象 |

### 7.5 竞争格局表字段

| 模块 | 字段 |
| --- | --- |
| 基础信息 | 店铺名、商品链接、主图、排名、销量、客单价 |
| 产品信息 | 款式、材质、厚度、尺寸、颜色、是否定制 |
| 价格信息 | 标价、券后价、SKU价格、价格带、活动价 |
| SKU信息 | SKU数量、主销SKU、SKU销售占比 |
| 卖点信息 | 主卖点、辅助卖点、详情页重点 |
| 视觉信息 | 1:1主图、3:4主图、场景图、文案结构 |
| 评价信息 | 好评点、差评点、问大家高频问题 |
| 流量信息 | 搜索、推荐、付费、直播、内容流量 |
| 客服信息 | 话术、响应、顾虑解决、推荐能力 |
| 我方机会 | 对手优势、对手弱点、我方突破点 |

### 7.6 判断标准

| 竞品现象 | 说明 | 我方动作 |
| --- | --- | --- |
| 竞品销量高但视觉差 | 产品需求成立，视觉有机会 | 做高点击主图 |
| 竞品销量高但差评多 | 需求成立，产品有痛点 | 做产品升级 |
| 竞品价格高但销量好 | 用户愿意为价值付费 | 做高价值利润款 |
| 竞品低价卖爆 | 价格敏感需求强 | 做引流款或供应链款 |
| 小店/C店涨得快 | 产品或趋势强 | 快速拆解跟进 |
| 竞品SKU结构单一 | 组合和价格梯度有机会 | 做SKU升级 |
| 竞品客服弱 | 转化承接有机会 | 优化客服话术 |

### 7.7 案例参考

拼接爬爬垫的例子：如果用户关心边条够不够，而市场很多拼接款不能随机发边条，那么“边条数量/是否随意赠送”就应该成为竞争格局表中的核心字段，因为这可能决定产品转化。

这个案例说明：

**竞品分析表不能套模板，必须根据类目的真实决策因素设计字段。**

## 流程8：跨平台趋势机会分析

完整参考文档：[《跨平台趋势机会判断标准》](https://alidocs.dingtalk.com/i/nodes/0eMKjyp813BMXdMai4dpp6jEVxAZB1Gv)

### 8.1 目的

淘系不是唯一趋势来源。很多爆款趋势会先在小红书、抖音、短视频、直播、内容平台出现，再反向带动淘宝搜索和成交。

### 8.2 数据来源

1.  小红书高赞笔记
    
2.  抖音短视频
    
3.  抖音电商罗盘
    
4.  抖音商品榜总榜
    
5.  抖音搜索榜
    
6.  抖音直播榜
    
7.  其他相关类目热销榜
    

### 8.3 执行动作

第一步：在小红书搜索类目关键词和场景关键词。  
第二步：筛选点赞500+以上内容。  
第三步：记录高频风格、场景、卖点、评论问题。  
第四步：在抖音看商品榜、搜索榜、直播榜。  
第五步：筛选1000+点赞视频或商品卡销量500+款式。  
第六步：判断能否迁移到淘宝产品。

### 8.4 跨平台趋势表

| 字段 | 说明 |
| --- | --- |
| 平台 | 小红书/抖音/其他 |
| 内容链接 | 原始内容 |
| 点赞/销量 | 热度证明 |
| 内容主题 | 为什么火 |
| 产品元素 | 材质、颜色、功能、风格 |
| 人群 | 谁在互动 |
| 场景 | 什么使用场景 |
| 评论需求 | 用户问什么 |
| 淘宝是否已有 | 有/少/没有 |
| 可迁移方式 | 产品/主图/标题/详情页/视频 |
| 机会等级 | 高/中/低 |

### 8.5 判断标准

| 平台现象 | 机会判断 | 动作 |
| --- | --- | --- |
| 小红书高赞，但淘宝供给少 | 内容趋势机会 | 快速开发或上架 |
| 抖音直播爆，但淘宝搜索刚起 | 搜索趋势前夜 | 先做标题和主图布局 |
| 抖音商品卡销量高 | 需求已验证 | 找供应链或同款升级 |
| 评论区反复问购买渠道 | 淘宝承接机会 | 快速建链接 |
| 其他类目元素爆火 | 跨类目创新机会 | 借鉴到本类目 |

### 8.6 案例参考

沙发垫案例中提到，跨平台爆火文章和视频往往能带动淘宝链接爆发，因此可以结合小红书、抖音热销内容，在沙发垫海量选品池中寻找机会款式上架。

## 流程9：机会判断与产品创新设计

完整文档：[《机会判断可落地判断标准》](https://alidocs.dingtalk.com/i/nodes/DnRL6jAJMGBNokNmsXDajXmDWyMoPYe1)

### 9.1 目的

把前面所有分析结果转化成“可开发的产品机会”。

这里要用《12种寻找机会的方法》里的核心判断：

**好产品 = 容易爆 + 有利润。**

机会来源主要有三类：

1.  对手没有做
    
2.  对手少
    
3.  对手弱
    

这也是文档中判断目标产品的核心标准。

### 9.2 机会判断模型

每一个机会都要经过 6 个判断：

| 判断维度 | 判断问题 |
| --- | --- |
| 需求 | 用户是否明确在搜、在问、在买？ |
| 增长 | 关键词、产品、店铺、平台内容是否增长？ |
| 竞争 | 对手多不多？强不强？ |
| 利润 | 价格带和成本是否能支撑？ |
| 供应链 | 我们能不能做出来？ |
| 差异化 | 我们凭什么赢？ |

### 9.3 四刀切机会

机会藏在细分市场，要通过细分人群、细分产品、细分场景、细分价格带来切入。

#### 第一刀：细分人群

| 动作 | 示例 |
| --- | --- |
| 找不同人群的需求差异 | 学生党、宝妈、老人、精致女生、轻熟龄女性 |
| 判断人群是否有搜索词 | 学生宿舍、儿童专用、老人防滑 |
| 判断人群是否有痛点 | 怕贵、怕不安全、怕不好看、怕麻烦 |
| 形成定位 | “军训防晒神器”“宝宝安全爬爬垫”“老人防滑浴室垫” |

#### 第二刀：细分产品

| 创新方式 | 示例 |
| --- | --- |
| 模仿 + 微创新 | 爆款款式做材质升级 |
| 功能升级 | 防水、防滑、A类面料、防晒、抗菌 |
| 组合创新 | A款 + B款 = AB款 |
| 打对手缺点 | 对手滑，我做防滑；对手有味，我做无味 |
| 改款升级 | 沙发垫 → 沙发巾 → 异形沙发垫 → 沙发笠 |
| 增加卖点 | 从普通款升级为高价值款 |

#### 第三刀：细分场景

| 动作 | 示例 |
| --- | --- |
| 同一产品换场景 | 家用、宿舍、办公室、户外、母婴 |
| 同一功能换人群 | 防滑给老人，防水给母婴，易清洗给懒人 |
| 同一材质换表达 | 冰丝 = 夏季凉感；羊羔绒 = 冬季保暖 |

#### 第四刀：细分价格带

| 动作 | 示例 |
| --- | --- |
| 高价市场做中价 | 高性价比切入 |
| 低价市场做高价 | 材质/风格/功能升级 |
| 中间价格带空白 | 做大众主推款 |
| 同款多价格带 | 基础款、升级款、豪华款 |

### 9.4 创新设计方向

总结了几类可落地的开发方向：面料季节性更替、功能用户需求点升级、两个爆款元素结合、沙发垫/沙发巾/沙发套款式变更、跨类目热销元素开发。

可沉淀为通用产品创新方法：

| 创新类型 | 方法 | 沙发垫案例 | 通用表达 |
| --- | --- | --- | --- |
| 季节创新 | 按季节换材质 | 夏季冰丝、冬季羊羔绒 | 季节需求驱动产品替换 |
| 功能创新 | 按痛点升级 | 防水隔尿、防滑升级、A类面料 | 高频痛点变功能 |
| 元素组合 | 两个爆款元素结合 | A款+B款=AB款 | 爆款元素重组 |
| 形态变更 | 同类目改形态 | 沙发垫→异形沙发垫→沙发笠 | 老需求新形态 |
| 跨类目借鉴 | 借其他类目爆点 | 床品、四件套、地垫、地毯 | 外部爆点内部创新 |

### 9.5 机会评分表

| 维度 | 分值 | 判断标准 |
| --- | --- | --- |
| 需求明确度 | 20 | 有关键词、有评价、有问大家、有内容讨论 |
| 增长趋势 | 20 | 搜索、销量、内容、竞品排名上涨 |
| 竞争强度 | 15 | 对手少、弱、小店能起来 |
| 利润空间 | 15 | 价格带能支撑毛利 |
| 供应链可行性 | 15 | 能快速开发、打样、上架 |
| 差异化强度 | 15 | 有明确不同于竞品的卖点 |

评分标准：

| 分数 | 判断 |
| --- | --- |
| 85分以上 | 优先立项开发 |
| 70-84分 | 小批量测试 |
| 60-69分 | 继续观察或补数据 |
| 60分以下 | 暂不开发 |

## 流程10：产品开发与链接规划落地

### 10.1 目的

市场洞察的最后一步必须落到产品和链接，不然就是无效分析。

《市场洞察报告》中的第六个技能就是产品开发链接规划表，目的就是把前面所有市场分析落到产品开发和链接规划上，让链接定位一目了然。

### 10.2 执行动作

第一步：从机会评分表中选择高优先级机会。  
第二步：填写产品开发立项表。  
第三步：明确产品定位、人群、价格、卖点、竞品、视觉方向。  
第四步：确定SKU结构。  
第五步：确定主图和详情页表达。  
第六步：确定测款指标。  
第七步：提交立项或进入上架测试。

### 10.3 产品开发链接规划表

| 模块 | 字段 |
| --- | --- |
| 基础信息 | 产品名称、类目、开发负责人、计划上架时间 |
| 机会来源 | 行业热销、关键词趋势、评价痛点、价格带空白、跨平台趋势 |
| 目标人群 | 谁买、为什么买、核心痛点 |
| 使用场景 | 在哪里用、什么时候用 |
| 产品定位 | 一句话定位 |
| 核心卖点 | 第一卖点、第二卖点、第三卖点 |
| 产品规格 | 材质、尺寸、颜色、功能、包装 |
| SKU规划 | 引流SKU、主推SKU、利润SKU |
| 价格规划 | 成本、售价、券后价、毛利率 |
| 竞品对标 | 直接竞品、学习竞品、防御竞品 |
| 差异化策略 | 产品差异、价格差异、视觉差异、服务差异 |
| 主图方向 | 第一点击理由、背景、构图、文案 |
| 详情页方向 | 痛点、卖点、证据、场景、信任 |
| 测试指标 | 点击率、收藏加购率、转化率、成交人数 |
| 结论 | 立项/测试/暂缓 |

### 10.4 链接定位公式

每一个链接必须用一句话讲清楚：

**给谁，在什么场景下，解决什么问题，凭什么比竞品更值得买。**

例如：

**给有现代轻奢装修风格的家庭用户，在客厅沙发使用场景下，提供一款半边绒材质、黑白条纹设计、四季可用、不易滑动的中价位沙发垫。**

# 七、适用子场景分别输出

## 子场景1：淘宝 / 天猫新品开发

### 适用情况

准备开发新品，但不知道做什么产品、什么价格、什么卖点、什么人群。

### 执行流程

1.  看行业前300，找主流产品结构。
    
2.  看关键词TOP300，找需求结构。
    
3.  看评价问大家，找痛点和升级点。
    
4.  看价格带，判断新品切入区间。
    
5.  看竞品，确定对标对象。
    
6.  做机会评分。
    
7.  输出新品开发立项表。
    

### 关键判断

新品不是凭感觉开发，而是要满足：

**有搜索需求 + 有热销验证 + 有痛点可升级 + 有价格带空间 + 有竞品可对标。**

## 子场景2：爆款机会挖掘

### 适用情况

想找下一个有机会打爆的产品。

### 执行流程

1.  看飙升大盘。
    
2.  看飙升关键词。
    
3.  看飙升产品。
    
4.  看飙升店铺。
    
5.  拆爆款基因。
    
6.  判断对手强弱。
    
7.  快速找供应链或做微创新。
    

### 关键判断

优先选择：

| 机会类型 | 判断 |
| --- | --- |
| 趋势词增长快 | 需求正在起来 |
| 小店也能卖爆 | 竞争壁垒不高 |
| 竞品视觉差 | 主图有突破机会 |
| 竞品差评集中 | 产品升级有机会 |
| 供给少 | 可以抢第一波红利 |

## 子场景3：类目市场分析

### 适用情况

进入新类目，或判断一个类目是否值得做。

### 执行流程

1.  明确类目边界。
    
2.  导出行业前300。
    
3.  统计产品类型、价格带、材质、风格、场景。
    
4.  拆关键词需求。
    
5.  看头部店铺结构。
    
6.  判断市场集中度和竞争强度。
    
7.  输出类目机会结论。
    

### 关键结论格式

这个类目是否值得做，要回答：

1.  体量够不够？
    
2.  增长是否存在？
    
3.  竞争是否过强？
    
4.  是否有细分人群？
    
5.  是否有细分场景？
    
6.  是否有价格带空白？
    
7.  我们有没有优势？

## 子场景6：价格带布局

### 适用情况

## 子场景7：产品升级与迭代

### 适用情况

已有老品，但增长乏力、转化不稳、差评集中。

### 执行流程

1.  分析自己链接评价和售后。
    
2.  分析竞品评价和问大家。
    
3.  找高频痛点。
    
4.  判断哪些痛点可以产品升级。
    
5.  判断哪些痛点可以视觉表达解决。
    
6.  判断哪些痛点可以客服话术解决。

### 关键判断

| 痛点类型 | 解决方式 |
| --- | --- |
| 产品硬伤 | 改材质、工艺、结构 |
| 认知不清 | 改主图、详情页 |
| 购买顾虑 | 改客服话术、承诺 |
| 使用误解 | 增加说明书、详情页解释 |
| 竞品没解决 | 做成核心差异卖点 |

# 八、最终总结

这套市场洞察方法论可以概括为一句话：

**从全域趋势中找方向，从行业数据中找主流，从关键词中找需求，从评价问大家中找痛点，从价格带中找利润，从竞品中找突破，最后落到产品开发和链接规划。**

完整执行逻辑是：

**看全域 → 看行业 → 看关键词 → 看用户反馈 → 看价格带 → 看竞品 → 找机会 → 做创新 → 立项开发 → 链接落地。**

真正可落地的市场洞察，最后必须输出三个东西：

1.  **机会判断**：这个机会为什么值得做？
    
2.  **产品方案**：具体做什么产品，什么材质、功能、风格、价格？
    
3.  **链接规划**：给谁卖，怎么表达，打哪个竞品，测什么指标？
    

最终标准不是“报告写得完整”，而是：

**运营看完知道做什么，产品看完知道怎么开发，设计看完知道怎么画主图，老板看完知道为什么值得投入。**
