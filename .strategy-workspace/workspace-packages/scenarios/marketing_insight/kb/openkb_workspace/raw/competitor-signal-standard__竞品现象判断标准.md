# 竞品现象判断标准

把「竞品现象 → 判断 → 我方动作」升级成**完整可落地判断标准**。核心是：竞品分析不能只看“感觉强不强”，必须看 **销量、增长、视觉、评价、价格、SKU、客服、店铺层级、流量结构** 这些证据。

---

# 竞品现象判断标准：完整可落地版

## 一、先定义竞品样本范围

竞品不能随便选，建议分三类竞品：

| 竞品类型 | 选择标准 | 用途 |
| --- | --- | --- |
| 直接竞品 | 同类目、同价格带、同人群、同场景、同产品形态 | 判断能不能直接打 |
| 学习竞品 | 比我们强，但产品/视觉/打法值得学习 | 拆爆款结构 |
| 弱竞品 | 小店/C店、视觉差、评价差，但销量不错 | 找突破机会 |

每次竞品分析至少看：

| 样本 | 数量 |
| --- | --- |
| 直接竞品 | 5-10个 |
| 学习竞品 | 3-5个 |
| 弱竞品 | 3-5个 |

# 二、竞品现象判断总表

| 竞品现象 | 说明 | 明确判断标准 | 我方动作 |
| --- | --- | --- | --- |
| 竞品销量高但视觉差 | 产品需求成立，视觉有机会 | 竞品支付买家数/销量进入行业TOP50；但主图存在以下任意3项：产品主体不清、卖点不突出、文案过多、场景不匹配、图片质感差、同屏差异弱、点击理由不明确；同时竞品CTR低于行业均值或主图同质化≥70% | 做高点击主图，用更清晰的品类识别、更强第一点击理由、更低认知负担切入 |
| 竞品销量高但差评多 | 需求成立，产品有痛点 | 竞品销量进入TOP50，且同一问题在中差评中占比≥10%；若≥15%为严重痛点，≥20%为产品硬伤；如果该痛点与主卖点冲突，优先级升为P0 | 做产品升级，围绕高频差评改材质、结构、尺寸、工艺、包装或服务 |
| 竞品价格高但销量好 | 用户愿意为价值付费 | 竞品价格高于行业中位价≥30%，但支付买家数仍进入TOP50，或GMV占比明显高于销量占比；评价中“品质/高级/安全/专业/设计/品牌”等正向反馈占比≥15% | 做高价值利润款，强化材质、功能、设计、品牌背书、检测证明 |
| 竞品低价卖爆 | 价格敏感需求强 | 竞品价格低于行业中位价≤70%，支付买家数进入TOP50，且评价/标题/主图中“便宜、划算、性价比、平替”相关表达占比≥15%；GMV占比低于销量占比 | 做引流款或供应链款，控制成本，用低价拉流量，不把它当核心利润款 |
| 小店/C店涨得快 | 产品或趋势强 | 小店/C店近30天排名提升≥20名，或支付买家数增长≥50%，且非强品牌、非大店资源驱动；相关关键词搜索增长≥20% | 快速拆解产品、主图、标题、价格、SKU，判断是否可快速跟进或微创新 |
| 竞品SKU结构单一 | 组合和价格梯度有机会 | 竞品SKU少于3个，或主销SKU占比≥70%；缺少低价引流SKU、中价主推SKU、高价利润SKU中的任意2类；评价/问大家中有尺寸、颜色、组合、规格需求占比≥10% | 做SKU升级，补尺寸、颜色、组合装、升级款，形成价格梯度 |
| 竞品客服弱 | 转化承接有机会 | 高频问大家占比≥10%，但客服回复不清楚、不专业、不统一；咨询后无法解决尺寸、材质、功能、售后等关键顾虑；竞品差评中服务问题占比≥8% | 优化客服话术，把高频顾虑前置到主图、详情页和客服SOP，提高询单转化 |

# 三、逐条拆解可执行标准

## 1. 竞品销量高但视觉差

### 判断逻辑

销量高说明这个产品需求已经被验证。  
视觉差说明它不是靠视觉赢，而是靠产品、价格、流量或趋势赢。  
如果我们能做出更高点击率主图，就有机会抢同类流量。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 支付买家数 / 销量 | 判断需求是否成立 |
| 行业排名 | 判断是否进入热销层 |
| 主图点击率CTR | 判断视觉效率 |
| 主图结构 | 判断表达是否清楚 |
| 主图同质化 | 判断是否有差异机会 |
| 主图文案数量 | 判断认知负担 |
| 产品主体占比 | 判断品类识别效率 |

### 明确判断标准

满足以下条件，可判断为“销量高但视觉差”：

| 判断项 | 参考值 |
| --- | --- |
| 支付买家数 / 销量排名 | 行业TOP50 |
| 主图CTR | 低于行业均值，或低于我方目标CTR |
| 主图文案信息点 | 超过5个，信息负担偏高 |
| 产品主体面积 | 低于画面40%，品类识别弱 |
| 同屏视觉同质化 | ≥70% |
| 第一点击理由 | 3秒内看不清 |
| 主图清晰度/质感 | 明显低于TOP10平均水平 |

### 我方动作

| 动作 | 执行标准 |
| --- | --- |
| 重做第一点击理由 | 一张主图只表达1个核心卖点 |
| 降低认知负担 | 主图核心元素控制在3-5个 |
| 强化产品主体 | 搜索主图产品主体建议占画面60%-80% |
| 做差异化背景 | 竞品都白底时，可测试轻场景/功能演示 |
| 做A/B测图 | 至少测试2-3版主图 |
| 验证指标 | CTR ≥ 行业均值1.2倍，或较原图提升≥20% |

### 结论模板

> 竞品A销量进入行业TOP50，说明该产品需求成立。但其主图信息点超过6个，产品主体占比不足40%，第一卖点不清楚，且同屏竞品视觉同质化达到70%以上，说明视觉存在突破空间。我方应优先做高点击主图，通过更清晰的主体、更强第一点击理由和更低认知负担抢点击。

## 2. 竞品销量高但差评多

### 判断逻辑

销量高说明用户愿意买。  
差评多说明产品没有完全满足用户。  
这类竞品最适合做“产品升级款”。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 竞品销量 | 判断需求成立 |
| 中差评内容 | 判断产品痛点 |
| 差评占比 | 判断痛点强度 |
| 追评 | 判断长期使用问题 |
| SKU差评 | 判断是否某个SKU有问题 |
| 问大家 | 判断购买前顾虑 |
| 我方供应链 | 判断能不能解决 |

### 明确判断标准

| 判断项 | 参考值 |
| --- | --- |
| 竞品销量排名 | 行业TOP50 |
| 同一差评问题占比 | ≥10%，痛点成立 |
| 同一差评问题占比 | ≥15%，严重痛点 |
| 同一差评问题占比 | ≥20%，产品硬伤 |
| 同一问题在TOP10竞品中出现 | ≥5个竞品，行业共性痛点 |
| 差评与主卖点冲突 | 直接升为P0问题 |
| 自家也存在该问题 | 必须优先整改 |

### 我方动作

| 痛点类型 | 动作 |
| --- | --- |
| 材质差 | 换材质、加厚、升级面料 |
| 不防滑/不防水 | 重新做功能结构 |
| 尺寸不准 | 增加尺寸适配表，优化SKU |
| 有味道 | 换材料、加检测、优化包装 |
| 掉色/变形 | 改工艺、做测试证明 |
| 发错/漏发 | 优化仓配和包装检查 |
| 服务差 | 建立售后承诺和客服话术 |

### 验证指标

| 指标 | 标准 |
| --- | --- |
| 新品差评率 | 低于竞品同类问题占比50%以上 |
| 高频痛点评价占比 | 明显下降 |
| 问大家顾虑 | 被主图/详情页提前解释 |
| 转化率 | 不低于竞品，最好高于竞品 |

### 结论模板

> 竞品B销量高，说明需求成立。但中差评中“容易滑动”占比达到18%，属于严重产品痛点。若我方能通过防滑底升级解决该问题，并在主图中做防滑演示，就可以用“解决竞品硬伤”的方式切入。

## 3. 竞品价格高但销量好

### 判断逻辑

价格高还能卖，说明用户不是只看便宜，而是愿意为某些价值付费。  
这种竞品适合拆“高价值卖点”，做利润款或中高价升级款。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 价格 / 客单价 | 判断是否高价 |
| 支付买家数 | 判断高价是否有人买 |
| GMV占比 | 判断金额贡献 |
| 评价好评点 | 判断用户为什么愿意买 |
| 主图/详情页 | 判断价值表达 |
| 品牌/材质/功能 | 判断溢价来源 |

### 明确判断标准

满足以下3项以上，可以判断为“高价但销量好”：

| 判断项 | 参考值 |
| --- | --- |
| 价格高于行业中位价 | ≥30% |
| 支付买家数排名 | 行业TOP50 |
| GMV占比 | 高于销量占比 |
| 好评中品质/材质/设计/安全相关占比 | ≥15% |
| 高价商品转化率 | 不低于行业均值 |
| 问大家中价格质疑少 | 说明价格接受度较好 |
| 复购/追评正向 | 说明价值被认可 |

### 我方动作

| 动作 | 执行标准 |
| --- | --- |
| 做利润款 | 价格可定在中高价或高价带 |
| 强化价值证明 | 材质、检测、工艺、设计、品牌背书 |
| 做详情页证据链 | 不只是说高级，要证明为什么高级 |
| 主图表达价值 | 少打低价，多打品质和效果 |
| 评价运营 | 引导用户评价品质、效果、设计 |
| SKU设计 | 基础款 + 升级款 + 高配款 |

### 结论模板

> 竞品C价格高于行业中位价35%，但销量仍进入TOP50，且GMV占比高于销量占比，评价中“材质好、高级感、安全”相关反馈占比达到19%。说明该类目存在愿意为品质和设计付费的人群，我方可以开发中高价利润款。

## 4. 竞品低价卖爆

### 判断逻辑

低价卖爆说明价格敏感人群大，低价可能是重要流量入口。  
但低价款不一定赚钱，要判断适合做引流还是供应链优势款。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 价格 | 是否显著低于行业 |
| 支付买家数 | 是否真实卖爆 |
| GMV占比 | 判断金额价值 |
| 毛利 | 判断是否可持续 |
| 主图表达 | 是否打低价/赠品/活动 |
| 评价 | 用户是否因为便宜购买 |
| 店铺供应链 | 判断是否有成本优势 |

### 明确判断标准

| 判断项 | 参考值 |
| --- | --- |
| 价格低于行业中位价 | ≤70% |
| 支付买家数排名 | 行业TOP50 |
| 支付买家数占比高，但GMV占比低 | 低价走量 |
| 评价中“便宜/划算/性价比”占比 | ≥15% |
| 主图中价格利益表达 | 明显突出 |
| 毛利率低于目标线 | 不适合作利润款 |
| 竞品活动频繁 | 可能靠促销拉动 |

### 我方动作

| 情况 | 动作 |
| --- | --- |
| 我方有供应链优势 | 可跟进低价供应链款 |
| 我方无成本优势 | 不硬卷低价 |
| 低价有流量但毛利低 | 做引流款 |
| 低价带动中高价SKU | 做组合销售 |
| 竞品靠低价但差评多 | 做中价升级款 |
| 低价同质化严重 | 用视觉/场景做差异 |

### 验证指标

| 指标 | 标准 |
| --- | --- |
| 低价款毛利率 | 不低于最低经营红线 |
| 低价款带动访客 | 访客增长明显 |
| 关联销售 | 能带动中高价SKU成交 |
| ROI | 不能长期亏损投放 |

### 结论模板

> 竞品D价格低于行业中位价40%，支付买家数进入TOP20，但GMV占比明显低于销量占比，且评价中“便宜、划算”占比超过20%。说明该链接主要吃价格敏感需求，适合作为引流参考，不适合作为利润款参考。

## 5. 小店/C店涨得快

### 判断逻辑

小店/C店没有大品牌、大预算、大店铺权重，如果还能快速增长，往往说明：  
产品趋势强、视觉打法有效、价格带有机会，或者需求刚起来。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 店铺层级 | 判断是否小店/C店 |
| 排名变化 | 判断增长速度 |
| 支付买家数增长 | 判断成交增长 |
| 关键词增长 | 判断需求增长 |
| 主图/标题 | 判断是否抓住趋势词 |
| 价格带 | 判断是否卡位成功 |
| 跨平台热度 | 判断是否外部趋势带动 |

### 明确判断标准

满足以下任意3项，判定为“高价值跟进信号”：

| 判断项 | 参考值 |
| --- | --- |
| 店铺为小店/C店或非头部品牌 | 成立 |
| 近30天排名提升 | ≥20名 |
| 近30天支付买家数增长 | ≥50% |
| 相关关键词搜索增长 | ≥20% |
| 小店进入行业TOP50 | 成立 |
| 同类产品TOP50中供给少 | 少于5个 |
| 抖音/小红书有同趋势内容 | 高热 |
| 竞品主图/产品可复制 | 成立 |

### 我方动作

| 动作 | 执行标准 |
| --- | --- |
| 快速拆解 | 拆标题、主图、价格、SKU、卖点 |
| 判断增长来源 | 是趋势、价格、视觉，还是活动 |
| 找供应链 | 能做则快速打样 |
| 做微创新 | 不直接抄，做材质/功能/场景/视觉升级 |
| 小批量测试 | 7-14天测点击、加购、转化 |
| 持续监控 | 每7天看排名和销量变化 |

### 结论模板

> 竞品E是小店，但近30天排名提升了28名，支付买家数增长52%，相关关键词搜索增长24%。这说明增长大概率不是品牌驱动，而是产品趋势或细分需求驱动。我方应快速拆解并做微创新测试。

---

## 6. 竞品SKU结构单一

### 判断逻辑

SKU结构单一说明竞品只承接了一部分需求。  
如果用户有更多尺寸、颜色、组合、价格梯度需求，我方可以通过SKU结构升级提升客单价和转化。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| SKU数量 | 判断结构丰富度 |
| SKU价格层级 | 判断是否有价格梯度 |
| SKU销售占比 | 判断是否过度依赖单一SKU |
| 问大家 | 判断用户是否问尺寸/规格/颜色 |
| 评价 | 判断是否有规格不满足 |
| 竞品组合装 | 判断是否有套装机会 |

### 明确判断标准

| 判断项 | 参考值 |
| --- | --- |
| SKU数量 | 少于3个，结构单一 |
| 主销SKU销售占比 | ≥70%，过度集中 |
| 缺少低/中/高价格梯度 | 成立 |
| 问大家中尺寸/颜色/组合问题占比 | ≥10% |
| 评价中“尺寸不合适/颜色少/不够用”占比 | ≥8% |
| 客单价低但组合需求明显 | 适合组合装 |
| 用户购买多个单品 | 适合套装化 |

### 我方动作

| 机会 | 动作 |
| --- | --- |
| 尺寸不足 | 增加核心尺寸SKU |
| 颜色不足 | 增加主流颜色，不盲目加长尾色 |
| 组合不足 | 做2件套、3件套、家庭装 |
| 价格单一 | 做基础款、升级款、高配款 |
| 客单价低 | 用组合装提升客单 |
| 主推不清 | 设定引流SKU、主推SKU、利润SKU |

### SKU升级原则

| SKU角色 | 作用 |
| --- | --- |
| 引流SKU | 拉低入门价格，获取点击 |
| 主推SKU | 承接最大成交需求 |
| 利润SKU | 提升客单价和毛利 |
| 组合SKU | 提升件单价和使用场景覆盖 |
| 升级SKU | 承接品质/功能人群 |

### 结论模板

> 竞品F只有2个SKU，主销SKU占比超过75%，且问大家中尺寸适配问题占比12%。说明竞品SKU结构无法充分承接用户需求。我方可以增加尺寸和组合装，并设置引流SKU、主推SKU和利润SKU，提升转化与客单价。

## 7. 竞品客服弱（需要竞品客服数据）

### 判断逻辑

客服弱说明竞品在转化承接环节有漏洞。  
如果产品差异不大，客服承接能力可能决定询单转化率和客单价。

### 看哪些数据

| 字段 | 判断作用 |
| --- | --- |
| 问大家高频问题 | 判断用户顾虑 |
| 客服响应速度 | 判断接待效率 |
| 客服回答质量 | 判断专业度 |
| 客服推荐能力 | 判断能否提高客单 |
| 差评中的服务问题 | 判断服务短板 |
| 询单转化率 | 判断客服结果 |

# 六、最终总结

竞品分析不是为了判断“谁强谁弱”，而是为了找我方切入口。

一句话标准：

**销量高看需求，差评多看产品升级，视觉差看点击机会，价格高看利润空间，低价爆看引流价值，小店涨看趋势机会，SKU少看结构机会，客服弱看转化机会。**
