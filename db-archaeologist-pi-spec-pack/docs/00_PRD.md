# PRD — DB Archaeologist Agent MVP

## 1. 背景

当前智能体数仓 API 文档包含大量历史接口、迁移接口、测试接口、公共 API、商品/关键词/竞争/投流/任务接口。问题不是接口数量少，而是：接口能力边界不清、字段语义不统一、质量状态不透明、Agent 无法稳定选工具。

因此需要 DB Archaeologist Agent，把 API 文档编译为 Agent 可读、可查、可选、可调用的资产层。

## 2. MVP 用户

- Agent 开发者：希望快速知道某个业务问题该调用哪些 API。
- 数据产品经理：希望维护 API 能力目录、字段含义和指标口径。
- 数据研发：希望发现接口缺文档、重复路径、空返回、乱码等问题。
- 业务分析 Agent：希望根据任务自动选择工具链。

## 3. 核心场景

### 3.1 API 问答

示例问题：

- “有没有查商品核心指标的接口？”
- “关键词趋势分析有哪些接口？”
- “哪个接口能查竞争格局 V3？”
- “哪些接口返回字段说明为空？”
- “商品诊断需要哪些接口组合？”

输出：

- 候选 API/Tool 列表。
- 方法与路径。
- 领域归属。
- 质量状态。
- 参数要求。
- 字段/指标摘要。
- 风险提示。

### 3.2 Agent 自动选工具

输入：

> 分析某个商品最近 7 天转化下降的原因。

输出：

- 推荐工具链：商品基础信息 → 核心指标 → 流量来源 → 付费推广 → SKU → 评论/主图。
- 参数缺口：goods_id、start_date、end_date、shop_id。
- 不建议使用的接口：空返回、字段缺失、测试接口。
- 解释：为什么需要这些工具。

## 4. MVP 范围

### In Scope

- 解析 Markdown API 文档。
- 创建 ApiAssetCard。
- Domain Mapping。
- API 质量评分。
- Tool Registry。
- Knowledge Graph triples。
- API 问答。
- Agent 自动选工具。
- Pi runtime custom tools 接入。

### Out of Scope

- 不接生产数据库。
- 不直接调用真实生产 API。
- 不做完整前端管理台。
- 不做自动修复 API。
- 不做复杂权限系统，只做策略字段和模拟校验。

## 5. 成功指标

- 95%+ 文档索引行可解析为 ApiAssetCard seed。
- P0/P1 接口 30 个以上可完成领域映射。
- 至少 10 个工具进入 Tool Registry seed。
- API 问答 top-3 命中率 ≥ 0.8。
- 自动选工具 case 通过率 ≥ 0.75。
- 空返回/字段缺失/重复路径接口必须被正确降级。
